# Visual Direction And Showcase

Use this when the user asks for a beautiful, premium, polished, designed, high-end, less generic, or style-matched deck, or when the source material does not imply a clear visual direction.

## Core Principle

When visual direction is unclear, do not guess one "nice" style. Offer three distinct directions, then build the chosen one into a reusable deck grammar.

Load `style-preset-library.md` when you need named presets or a base-plus-modifier style decision.

## Three-Direction Sprint

Create three short art directions:

```markdown
## Direction A: <name>
Best for:
Visual language:
Typography:
Palette:
Image/diagram treatment:
Risks:

## Direction B: <name>
...

## Direction C: <name>
...
```

Each direction must differ in more than color. Vary typography, composition, information density, image treatment, and page rhythm.

Recommended families:

| Family | Use For | Signals |
| --- | --- | --- |
| Executive editorial | strategy, keynote, founder vision | large claims, serif display, restrained pages |
| Consulting precision | board, procurement, enterprise sales | tight grid, evidence, clear hierarchy |
| Product launch | product story, feature release | hero imagery, benefit moments, demo flow |
| Technical atlas | AI platforms, architecture, research | diagrams, scorecards, traceable logic |
| Magazine casebook | customer stories, industry cases | spreads, captions, proof pages |
| Cultural ink-tech | Chinese technology/report tone | warm paper, restrained ink motif, modern grid |
| Interface-native | SaaS, workflow, product UI | screenshots, browser frames, UI rhythm |

## Showcase Pass

After choosing or recommending a direction, create a two-page showcase:

- Page 1: cover, thesis, or section divider
- Page 2: dense content page such as case, diagram, data, or comparison

The showcase is successful only if:

- the deck has a recognizable point of view
- the dense page stays readable
- the two pages clearly belong together
- the style can support 10+ slides without becoming repetitive

## Anti-Generic Rules

Avoid:

- default purple/blue AI gradients
- decorative blobs, fake aurora, empty neon grids
- emoji as a visual system
- generic robot or handshake imagery
- repeated bento cards with swapped icons
- stock people pointing at transparent screens
- unreadable tiny labels used to look "technical"

Prefer:

- real product screenshots, logos, diagrams, charts, and source artifacts
- one strong page motif repeated subtly
- editorial typography with clear hierarchy
- meaningful image crops and captions
- large evidence objects instead of many small cards
- atmosphere at the edge, meaning in the center

## Showcase Library

When maintaining this skill, keep examples under:

```text
assets/showcases/ppt/
```

Each example should include:

- `index.html` or a preview image
- the intended deck type
- what visual grammar it demonstrates
- what not to copy blindly

Do not add showcase pages that are only decorative. Every showcase should teach a reusable deck move.
