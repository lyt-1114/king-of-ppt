# Writing Style Guide

Use this when a presentation needs richer Chinese copy, more elegant wording, fuller slide content, or report-style readout language.

## Copy Layer Before Layout

Every important slide should have four layers:

1. Thesis: one complete sentence that states the slide's point of view.
2. Support: 2-4 content blocks that explain mechanism, evidence, contrast, case, or implication.
3. Proof / texture: a metric, example, workflow step, source, artifact, quote, or scenario detail.
4. Core value line: a short bottom-line sentence for report-style decks.

Avoid slides that only contain nouns, tags, and icons. A deck can look full but still feel empty if the writing has no argument.

## Elegant Chinese Slide Copy

Aim for precise, composed, business-readable Chinese:

- Use complete phrases: "把审核、补件、复核接回同一条可追踪链路" is better than "审核链路优化".
- Prefer concrete verbs: 接入、沉淀、回流、压缩、识别、复核、追踪、校准、联动.
- Keep poetic language in headlines or value lines; keep evidence pages precise.
- Use parallel structures for rhythm: "先接稳数据，再接顺流程，最后让 Agent 进入受控执行."
- Let Chinese cultural motifs support the message: "汇流" for integration, "回环" for feedback, "留痕" for auditability.
- Do not use vague hype: "赋能未来、智慧升级、全面领先" unless followed by concrete mechanism.

## Slide Content Minimums

Use these minimums unless the slide is a cover, divider, or closing page:

- Analysis slide: 1 thesis + 3-4 causes or symptoms + 1 implication.
- Architecture slide: 1 overview sentence + layer/module labels + inputs/outputs + 1 value line.
- Case slide: situation + intervention + result + reusable asset.
- Comparison slide: comparison basis + 4-6 rows + decisive takeaway.
- Evaluation slide: dimensions + method + example metric + how results feed improvement.
- Roadmap slide: phases + deliverables + decision gate + preparation list.

If a slide does not meet the minimum, either enrich it or merge it into another slide.

## Case Writing Template

Use this pattern for substantial case pages:

```text
Situation:
客户原本卡在 [具体流程]，问题不是单点效率，而是 [跨角色/跨系统/质量/风险] 无法统一。

Intervention:
WanFlow 将 [数据/规则/状态/Agent/人工复核] 接成一条执行链，让 [角色] 在同一视图下推进。

Result:
上线后 [指标] 改善，同时沉淀 [可复用资产]，后续可复制到 [相邻场景]。
```

Compress the template into slide copy, but keep all three ideas present.

## Richness Without Overcrowding

Make slides feel substantial by improving information design, not by shrinking text.

- Add micro-explanations under module labels.
- Add "why it matters" captions to metrics.
- Add "final asset" rows to case pages.
- Add small flow steps instead of long paragraphs.
- Use notes for detail that supports readout but does not need to be on the slide.
- Add appendix pages for dense evidence if the main story needs to breathe.

## Headline Patterns

Good headlines make a claim:

- "企业 AI 落地的瓶颈，不在模型，而在执行链"
- "从点状工具到运营系统，关键差异是可追踪与可复盘"
- "案例价值不止是提效，更是沉淀可复制的业务资产"
- "先用一条链路证明价值，再复制到更多业务入口"

Weak headlines only name a topic:

- "项目背景"
- "解决方案"
- "案例展示"
- "技术架构"

Topic labels can remain as small section markers, but the main title should carry a point of view.

## Bottom Value Line

For Chinese report-style decks, add a bottom value line on content pages:

```text
核心价值：把原本分散的业务动作接成一条可追踪、可复核、可持续优化的执行链。
```

Keep it under 28 Chinese characters when possible. It should summarize the page, not introduce a new idea.
