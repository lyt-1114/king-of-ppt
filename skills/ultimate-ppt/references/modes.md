# Deck Modes

## Executive Consulting

Use for board, enterprise sales, strategy, procurement, and serious B2B decisions.

Spine: Problem -> Diagnosis -> Solution architecture -> Proof -> Rollout -> Decision.

Style: restrained, precise, evidence-led. Use diagrams, metrics, rollout tables, and decision pages.

## Pitch / Product Launch

Use for product introductions, fundraising, sales conversations, and demo days.

Spine: Problem -> Why now -> Product -> Use cases -> Proof -> Offer -> CTA.

Style: direct, modular, high momentum. Use benefits, product moments, customer proof, and clear CTA.

## Editorial Keynote

Use for talks that need memorability, founder vision, thought leadership, or emotional narrative.

Spine: Hook -> Context -> Core idea -> Field scenes -> Shift -> Takeaway.

Style: magazine-like, cinematic, sparse, rhythmic. Use big quotes, chapter dividers, field notes, and strong contrast.

## Browser-Native Interactive

Use when the HTML experience matters: animation, keyboard navigation, single-file sharing, or modern visual demos.

Spine: Interface metaphor -> Live layers -> Scenario dashboards -> Signals -> Trust controls -> CTA.

Style: distinctive, responsive, no scroll. Every slide fits 100vh and carries one action.

## Visual System / Prompt Board

Use when the real need is image direction, design language, or reusable slide assets.

Spine: Principles -> Prompt boards -> Asset recipes -> Guardrails -> Handoff.

Style: art-direction board. Include positive prompts, negative prompts, composition rules, and usage examples.

## Training / Course

Use for workshops, internal enablement, tutorials, or teaching.

Spine: Outcome -> Mental model -> Worked example -> Practice -> Common mistakes -> Recap.

Style: clear, progressive, learner-paced. Include exercises and speaker notes.
